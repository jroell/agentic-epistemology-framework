\begin{abstract}
Autonomous agents and synthetic personas increasingly act from long-lived context rather than single-turn instructions. Their failures are therefore epistemic: the system may overweight the wrong evidence, ignore a persona constraint, express confidence unsupported by its latent state, or provide a plausible but unfaithful justification. This paper introduces the Agentic Epistemology Framework (\method), an audit architecture for representing and tracing beliefs, confidence, justifications, frames, and facet-conditioned persona constraints. The framework combines structured observer traces with black-box faithfulness tests, confidence calibration checks, and optional representation-level probes inspired by recent work on chain-of-thought faithfulness, model self-knowledge, representation engineering, and sparse autoencoder interpretability. We extend the framework with \emph{facets}: persistent psychological, socio-ecological, psychographic, and interaction constraints that compose nonlinearly into an agent's active frame. A TypeScript reference implementation includes automated benchmarks for root-cause diagnosis and facet-frame faithfulness. In controlled synthetic cases, full epistemic traces recover injected generic root causes with 1.000 accuracy and recover injected facet-faithfulness failures with 0.665 accuracy, while ablated traces degrade substantially. These results support the central claim that explicit epistemic state and facet-aware traces make agent behavior more diagnosable, while leaving open the harder problem of validating open-ended real-world simulations.
\end{abstract}

\section{Introduction}
Autonomous agents now plan, use tools, communicate, and revise internal state across long-running interactions. Synthetic personas add another layer: they must remain faithful to a constructed identity while responding naturally to novel stimuli. In both cases, important causal factors are often hidden inside prompt text, model activations, unstructured logs, or generated explanations that may not faithfully describe why the model produced a response.

This paper introduces the Agentic Epistemology Framework (\method), a structured architecture for auditing agent reasoning. \method{} represents beliefs as propositions with confidence and justifications, records observer events for epistemic transitions, and models frames as active cognitive perspectives that influence evidence weighting, source trust, and action thresholds. The paper further extends frames with \emph{facets}: typed persona constraints that jointly shape the active frame. Facets can represent psychological dispositions, lived-context constraints, psychographic motivations, and interaction style. In a synthetic human simulation, facets are not independent prompt fragments. They interact nonlinearly, and those interactions should be inspectable.

The first publication of \method{} has two goals. First, it defines a vocabulary and implementation pattern for epistemic auditability in agents and synthetic societies. Second, it provides executable benchmarks that test whether the resulting traces make known failure causes diagnosable. The framework does not claim that generated persona behavior is valid in the world merely because the system produces plausible text. Instead, it asks whether the system can expose enough structured evidence to debug when a persona fails to follow its own construction.

\subsection{Contributions}
This paper makes four contributions:
\begin{enumerate}[leftmargin=*]
  \item A structured epistemic ontology for beliefs, confidence, justifications, frames, facets, perceptions, actions, and observer events.
  \item A facet composition model in which persistent persona constraints contribute nonlinearly to an agent's active frame.
  \item A TypeScript reference implementation with automated root-cause and facet-faithfulness benchmarks.
  \item An evaluation protocol that combines structured traces with confidence calibration, black-box perturbation tests, and optional representation-level probes for reasoning faithfulness.
\end{enumerate}

\section{Background and Motivation}
\method{} draws from belief-desire-intention models \citep{bratman1987intention, rao1991modeling, wooldridge2009multiagent}, epistemic logic \citep{fagin2003reasoning}, belief revision \citep{alchourron1985logic, gardenfors1988knowledge}, frame semantics \citep{minsky1974framework, fillmore1976frame, lakoff1987women}, argumentation theory \citep{dung1995acceptability, nute1994defeasible}, and agent-based modeling \citep{epstein1996growing}. These traditions provide useful structure, but current LLM-based agents create additional audit problems.

Recent work shows that natural-language explanations should not be treated as automatically faithful. Chain-of-thought explanations can systematically misrepresent the true reasons for a model's answer \citep{arxiv230504388}, and faithfulness requires intervention-style tests rather than surface plausibility alone \citep{arxiv230713702}. Calibration and self-evaluation research shows that language models can sometimes estimate whether their own answers are correct, but this depends strongly on elicitation format and task setting \citep{arxiv220705221}. Black-box hallucination detection methods, such as sampling-based consistency checks, provide another way to test whether outputs are stable under repeated generation \citep{arxiv230308896}. Meanwhile, representation engineering and sparse autoencoder work provide tools for probing latent directions or interpretable features when model activations are available \citep{arxiv231001405, arxiv230908600}.

\method{} treats these lines of work as complementary. Observer traces expose the explicit state the system is designed to use. Black-box perturbations test whether outputs change when facets, evidence, or prompts are altered. Calibration tests ask whether the system knows when its response is likely faithful. Representation-level probes, when available, can test whether latent activations align with the asserted frame or facet. The framework is designed to hold all of these evidence types in a single audit trail.

\section{Facet-Conditioned Synthetic Personas}
Synthetic human simulation treats a persona as a complex adaptive system rather than a flat list of demographic labels. A persona has internal psychological dispositions, lived-context constraints, motivations, trust heuristics, resource constraints, and linguistic interaction style. In the Vurvey implementation inspected for this work, these variables are represented as a graph of PM facets. A facet is an axis of variation such as income, response style, source trust, or domain expertise. A facet value contains stable identifiers, prompts, taxonomy metadata, and reliability fields. Personas link to selected values through persona fragments. Population molds define facet distributions, mandatory and preferred values, weights, selection modes, and constraints.

This implementation suggests a natural extension to \method. A \textbf{facet} is a typed, persistent constraint with fields:
\begin{equation}
Facet_i = (code_i, value_i, domain_i, prompt_i, weight_i, confidence_i, provenance_i).
\end{equation}
The active frame is composed from the current stimulus $x_t$, the persona state $S_t$, and selected facets $Facet_{1:n}$:
\begin{equation}
F_t = g(S_t, x_t, Facet_{1:n}, I_{1:n}),
\end{equation}
where $I_{1:n}$ denotes interaction terms between facets. This form is intentionally nonlinear. For example, high extraversion can produce very different behavior when combined with fragile self-confidence and extreme time poverty than when combined with high perceived control and abundant resources.

\subsection{Facet Domains}
The framework organizes facets into four audit domains:
\begin{enumerate}[leftmargin=*]
  \item \textbf{Psychological facets}: stable traits, anxiety level, self-confidence archetype, and emotional regulation constraints.
  \item \textbf{Socio-ecological facets}: locale, life stage, socioeconomic status, time poverty, family structure, and other lived-context constraints.
  \item \textbf{Psychographic facets}: motivations, adoption-curve position, source trust, perceived behavioral control, and decision heuristics.
  \item \textbf{Interaction facets}: response style, conversational friction, objection tendency, and linguistic modulation.
\end{enumerate}

Each facet should be auditable at three levels: whether it was selected, whether its prompt or structured representation reached the generation path, and whether the final response reflects its expected influence.

\section{Epistemic State and Operational Rules}
A \textbf{belief} is a proposition $P$ held by an agent with confidence $conf \in [0,1]$ and a justification. A \textbf{justification} is evidence, reasoning trace, source, or derivation path supporting a belief or response. A \textbf{frame} is the active cognitive lens that modulates interpretation and evidence weighting. A \textbf{facet} is a persistent constraint that contributes to frame composition. An \textbf{observer event} is a structured record of an epistemic transition.

\method{} is guided by five operational rules:
\begin{enumerate}[leftmargin=*]
  \item \textbf{Traceability}: important actions and responses should trace back to beliefs, justifications, frames, and contributing facets.
  \item \textbf{Explicit uncertainty}: beliefs and evaluator judgments should expose confidence rather than binary assertions.
  \item \textbf{Frame awareness}: agents should record when frames change evidence weighting, source trust, or interpretation.
  \item \textbf{Facet faithfulness}: persona responses should expose which facets were expected to constrain the response and which facets were actually expressed.
  \item \textbf{Intervention readiness}: traces should support counterfactual tests such as removing a facet, swapping a facet value, or perturbing evidence source trust.
\end{enumerate}

\section{Confidence, Frame, and Facet Updates}
Let $conf_{old}$ denote an agent's current confidence in proposition $P$, and let $e$ denote new evidence relevant to $P$. \method{} uses update functions of the form:
\begin{equation}
conf_{new} = f(conf_{old}, just_{old}, e, F_t).
\end{equation}

For frame-weighted evidence, evidence $e$ suggests a target confidence $C(e,P) \in [0,1]$. The active frame assigns evidence weight $w_{F_t}(e)$:
\begin{equation}
conf_{new} = (1 - w_{F_t}(e)) \cdot conf_{old} + w_{F_t}(e) \cdot C(e,P).
\end{equation}

For facet-conditioned personas, $w_{F_t}(e)$ can be decomposed into facet contributions:
\begin{equation}
w_{F_t}(e) = \sigma\left(b + \sum_i \alpha_i h(Facet_i, e) + \sum_{i<j} \beta_{ij} h(Facet_i, Facet_j, e)\right),
\end{equation}
where $h$ maps facets and evidence to contribution scores, $\alpha_i$ are individual facet weights, $\beta_{ij}$ are interaction weights, and $\sigma$ bounds the resulting saliency. The observer should log individual contributions and interaction terms whenever the implementation can expose them.

\section{Auditing Faithfulness}
\method{} distinguishes four kinds of faithfulness evidence.

\textbf{Trace faithfulness} asks whether the system recorded the inputs and transformations it was designed to use. For synthetic personas, this includes selected facets, facet values, prompt fragments, weights, constraints, and response markers.

\textbf{Behavioral faithfulness} asks whether the output reflects the selected persona constraints. This can be measured by entailment checks, contradiction checks, response-style classifiers, and facet-marker coverage.

\textbf{Counterfactual faithfulness} asks whether changing an input that should matter changes the output in the expected direction. Examples include swapping financial outlook from crisis flow to abundance, changing response style from declarer to verbatim repeater, or replacing peer-trust with institutional-trust.

\textbf{Latent faithfulness} asks whether hidden representations align with the claimed frame or facet when activations are available. Representation engineering can test directions corresponding to persona-relevant concepts, and sparse autoencoders can expose interpretable features. These probes are optional because many production systems expose only black-box access, but the framework leaves room for them as stronger evidence.

\section{Reference Implementation}
The reference implementation is written in TypeScript and is publicly available at \url{https://github.com/jroell/agentic-epistemology-framework}. It implements beliefs, justifications, frames, agents, memory, actions, tools, and observer events. The implementation now includes two benchmark harnesses: a generic root-cause benchmark and a facet-frame faithfulness benchmark.

The Vurvey production codebase informed the facet extension. Its PM facet graph includes global facets and versioned facet values, `headerPrompt` and `detailPrompt` fields used to build persona prompts, `PmPersonaFragment` rows that link personas to selected facet values, mold facet configurations with weights and options, and auto-configuration services that recommend and weight facet distributions for population molds. These implementation details motivate the audit requirement: a persona response should be diagnosable back to the facet graph, prompt fragments, selected values, and composed frame that produced it.

\section{Automated Root-Cause Benchmark}
The generic benchmark asks whether structured epistemic traces make it possible to identify the cause of agent failures in scenarios where the ground-truth cause is known. It covers four injected causes: frame misweighting, source-trust error, threshold miscalibration, and justification gap. It evaluates four trace conditions: full trace, no-frame trace, no-justification trace, and event-only trace.

We ran the benchmark with 160 synthetic cases using seed 42. The benchmark command was:
\begin{verbatim}
npx ts-node --transpile-only examples/root-cause-benchmark.ts --seed=42 --cases=160
\end{verbatim}

\section{Facet-Frame Faithfulness Benchmark}
The second benchmark targets synthetic persona auditing. Each case constructs a persona from six facets: anxiety level, time poverty, financial outlook, source trust, response style, and adoption-curve position. The harness injects one of five failure types:
\begin{enumerate}[leftmargin=*]
  \item \textbf{Facet omission}: a required facet is absent from the composed frame.
  \item \textbf{Facet misweighting}: a facet contribution is materially lower or higher than its expected calibration.
  \item \textbf{Facet conflict}: two facets conflict and the response follows one while ignoring the other.
  \item \textbf{Persona drift}: the response introduces an identity inconsistent with the facet set.
  \item \textbf{Response-style violation}: the response ignores the selected interaction-style facet.
\end{enumerate}

The benchmark evaluates four conditions: full trace, no facet weights, no facet prompts, and event-only trace. We ran 200 cases with seed 42:
\begin{verbatim}
npx ts-node --transpile-only examples/facet-frame-faithfulness-benchmark.ts --seed=42 --cases=200
\end{verbatim}

\section{Facet Faithfulness Audit Service Experiment}
The benchmark above motivated a concrete implementation: \texttt{FacetFaithfulnessAuditService}. The service takes a persona response, selected facets, rendered prompt provenance, model metadata, self-reported confidence, and optional counterfactual probes. It returns a facet influence ledger, marker coverage, contradictions, counterfactual sensitivity, calibration score, final faithfulness score, primary diagnosis, causal facets, evidence, and recommended debug actions.

The service implements the audit hooks proposed by the framework: selected facet IDs and values, taxonomy versions, prompt hashes, configured and runtime weights, frame composition, suppressed facets, inferred interaction terms, expected markers, observed markers, forbidden marker contradictions, confidence calibration, and counterfactual sensitivity. A Vurvey adapter maps PM facet values, facet codes, \texttt{headerPrompt}, \texttt{detailPrompt}, confidence scores, taxonomy versions, and configured weights into this audit input shape.

We evaluated the audit service on eight hand-constructed Vurvey-style persona cases covering faithful responses, prompt omission, facet misweighting, facet conflict, persona drift, response-style violation, and low counterfactual sensitivity. The goal of this experiment is not to prove external predictive validity. It tests whether the implemented service can recover failure causes from increasingly rich evidence conditions.

The experiment exposed an important design lesson. Mean faithfulness score is not the same as diagnostic accuracy. The full audit condition intentionally surfaces more missing markers and calibration penalties than the weaker conditions, so the mean score can remain conservative even while diagnosis improves. This is desirable for an audit system: it should not inflate trust simply because it sees more evidence. The useful output is the diagnosis and debug action, not a single high score.

In the faithful fixture, the ledger recorded five selected facets, prompt inclusion for each facet, prompt hashes, configured and runtime weights, and interaction terms such as \texttt{financial\_outlook:crisis\_flow x adoption\_curve\_position:innovator}. The service correctly marked the response as faithful while still listing missing secondary markers, such as explicit novelty language. In the prompt omission fixture, the rendered prompt omitted \texttt{financial\_outlook}; the service diagnosed \texttt{facet\_prompt\_omission} and recommended verifying prompt assembly. In the persona drift fixture, the response introduced an unsupported enterprise CTO identity with unlimited budget; the service diagnosed \texttt{persona\_drift} and recommended regenerating or constraining the persona biography.

\section{Discussion}
The generic benchmark shows that structured epistemic traces can recover simple injected failures when the trace contains the relevant fields. The facet benchmark is more realistic and harder. Full traces recover many failures, but not all. This is useful: persona faithfulness cannot be reduced to checking that a prompt contains the right words. It requires tracing facet selection, prompt rendering, frame composition, expected behavioral markers, response content, and sometimes counterfactual or latent evidence.

For Vurvey-style synthetic personas, \method{} suggests several concrete audit hooks:
\begin{enumerate}[leftmargin=*]
  \item Record the selected facet IDs, value IDs, codes, taxonomy versions, and prompt hashes used for a response.
  \item Record facet weights and interaction terms used to compose the active frame.
  \item Record expected response markers derived from selected facets, such as cost objection, peer evidence request, time constraint, or response-style marker.
  \item Run counterfactual probes that swap one high-impact facet while holding the rest constant.
  \item Track self-rated confidence and compare it with evaluator judgments and response consistency.
  \item When model internals are available, add representation-level probes for persona-relevant directions or sparse features.
\end{enumerate}

\section{Limitations}
The benchmarks use controlled synthetic failures with known labels. They establish that the implementation can generate useful diagnostic traces, not that all synthetic personas are valid or predictive in deployment. The facet benchmark uses deterministic heuristics for diagnosis and should be extended with noisy traces, human debugging studies, live product prompt paths, and held-out response data.

The framework also depends on the quality of facet definitions, prompt fragments, weights, and constraints. If the facet graph itself encodes weak theory or biased assumptions, \method{} will make those assumptions more visible but will not make them correct. Detailed observer logs can expose sensitive persona or respondent data, so production use requires careful retention, access control, and redaction policy.

\section{Conclusion}
The Agentic Epistemology Framework provides an audit architecture for autonomous agents and synthetic personas. By making beliefs, confidence, justifications, frames, facets, and observer events first-class constructs, \method{} supports debugging not only what an agent said, but why the system should have expected that behavior from its configured epistemic state. The facet extension connects this audit layer to synthetic human simulation: facets become inspectable contributors to frame composition, and persona faithfulness becomes a measurable, diagnosable property rather than a matter of surface plausibility.
